# auto-pilot — 무인 자율 태스크 실행 스킬

이 스킬은 복잡하거나 장시간이 소요되는 작업에 대해 사용자의 중간 개입 및 질문 없이 에이전트가 100% 자율적으로 판단하여 완수하기 위한 지침입니다.

## 기본 철칙
1. **질문 금지**: 중간 단계에서 선택지나 설계 방향을 묻지 않습니다. 보편적이고 가장 안전한 최선의 선택안을 에이전트가 스스로 선택해 수행합니다.
2. **자율적 디버깅 및 에러 복구**: 명령 실행이나 빌드 중 오류가 나면 사용자에게 보고하지 않고, 오류 로그를 스스로 분석한 뒤 코드를 수정하거나 설정을 변경하여 통과할 때까지 반복 수행합니다.
3. **완벽한 사전 검증**: 작업의 최종 결과를 테스트 및 컴파일(빌드) 검증을 거쳐 에러가 없을 때만 완료로 판정합니다.

## 실행 절차

### 1. 목표 및 계획 수립
- 사용자의 프롬프트 요구사항을 세부 Task 리스트로 쪼갭니다.
- 계획을 `memory/06-state/plans` 하위에 작성하거나 컨텍스트 상에서 수립합니다.

### 2. 단계별 자율 수행 (Loop)
- 각 Task에 대해 필요한 코드를 생성하거나 변경합니다.
- `run_command` 또는 빌드 툴을 활용하여 즉시 정적 검사 및 빌드 테스트(`pnpm build`, `go build` 등)를 수행합니다.
- 실패 시 에러 내용을 바탕으로 코드를 자동으로 수정(Self-correction)합니다.

### 3. 최종 무결성 검증
- 모든 Task 완료 시 전체 시스템의 헬스체크나 e2e 테스트 등을 작동하여 정합성을 교차 검증합니다.

### 4. 사후 보고 및 인계
- 모든 과정이 에러 없이 완전히 끝난 후에만 사용자에게 최종 완료 요약 보고서와 갱신된 Handoff(인계) 문서를 출력하여 전달합니다.
