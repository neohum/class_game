# 디자인 언어: 브루탈리즘 / 표현주의

aesthetic-system 보조 레퍼런스. director가 이 언어를 골랐을 때 로드한다.

## 정신

웹 브루탈리즘. **날것, 대담함, 규칙 위반의 의도성.** 매끈한 SaaS 미감에 대한 반동. 강한 대비, 두꺼운 보더, 하드 섀도, 시스템 폰트의 노골적 사용, 비대칭과 충돌을 무기로 강한 개성과 임팩트를 만든다. 단, **무질서가 아니라 의도된 거칠음**이다 — 골격(grid)은 여전히 단단해야 한다.

## 색

- **고대비, 대담한 색.** 순흑/순백 + 강렬한 원색(전자적 노랑·핫핑크·코발트). 중간톤을 피하고 극단을 쓴다.
- 보색·충돌색을 의도적으로 병치. 채도 높게.
- 그라데이션보다 **단색 블록**. 색면을 무기로.

## 타이포

- 시스템 폰트(Times, Courier, Arial)나 극단적 디스플레이 서체를 노골적으로.
- 거대한 타이포 대비: 초대형 헤드라인 + 작은 본문. 모노스페이스 적극 사용.
- 자간/행간을 일부러 타이트하거나 과장.

## 레이아웃·디테일

- **두꺼운 보더(2~4px+) 블랙.** border-radius 0이 기본.
- **하드 섀도**: blur 0의 오프셋 그림자(4px 4px 0 #000 류). 입체가 아니라 도장처럼.
- 비대칭·겹침·의도된 충돌. 그리드를 쓰되 일부러 깨는 포인트를 둔다.
- 밑줄·박스 등 노골적 UI 시그널.

## 모션

- 즉각적·딱딱한 전환 또는 과장된 모션. 스냅. 미묘함보다 존재감.

## 균형 잡기 (중요)

브루탈리즘은 접근성과 충돌하기 쉽다. **대비는 오히려 유리**(고대비)하지만, 가독성·포커스 표시·터치 타깃은 지킨다. critic의 WCAG 검증을 반드시 통과시킨다 — 거칠음은 스타일이고, 못 읽는 건 결함이다.

## 피해야 할 것

- 미묘한 soft shadow, 파스텔, 둥근 모서리, 절제된 그라데이션 — 이 언어의 정반대다. 또한 골격 없는 진짜 무질서(랜덤 배치)는 임팩트가 아니라 아마추어다.
