# Aesthetic System — 골격 위에 입히는 미감

geometry가 만든 수학적 골격은 차갑다. 여기에 색·타이포·위계의 살을 입혀 '아름답게 느껴지게' 하는 것이 이 스킬의 일이다. 단, 임의의 취향이 아니라 **체계화된 미감**을 만든다.

## 설계 순서

1. **언어 확인** — director가 고른 디자인 언어를 받고, 해당 `references/{언어}.md`를 로드해 어휘를 맞춘다.
2. **색 시스템** — 명도(L) 곡선 위에 팔레트를 배치한다 (아래).
3. **타이포 페어링** — 디스플레이/본문/모노 폰트 선택과 굵기·자간 운용 (스케일은 geometry 소유).
4. **시각 위계** — 대비 전략으로 주목 순서 설계.
5. **무드·모션 톤** — 언어의 정서를 그림자·모션·질감으로 구현.

## 색은 명도로 사고한다 (핵심)

임의 hex를 고르면 단계가 들쭉날쭉하고 접근성이 예측 불가다. **OKLCH**(지각적 균일 색공간)로 사고하라:

- **L (명도, 0~1)**: 텍스트/배경 대비를 좌우. 팔레트 단계를 L의 등간격으로 깔면 균일한 계단이 된다.
- **C (채도)**: 0이면 무채색. 액센트만 높이고 배경은 낮춰 눈의 피로를 줄인다.
- **H (색상)**: 색조. 같은 H에서 L만 바꿔 한 색의 명암 단계(50~950)를 만든다.

권장 토큰 구조(원시 → 의미):

```
neutral.{0,50,100,...,900,950}   ← L 등간격 회색 계단
brand.{50..950}                  ← 단일 H, L 계단
semantic: success/warning/danger/info (각 H, L 계단)
─ 의미 매핑 ─
surface.base / surface.raised / surface.sunken   (명도 1~4% 차로 깊이)
text.primary / text.secondary / text.muted        (대비 충족하는 L)
border.subtle / border.strong
accent.default / accent.hover / accent.active
```

## 다크모드 = 의미 계층 재매핑

원시 팔레트는 그대로 두고, **의미 토큰의 참조만 뒤집는다**. `text.primary`가 라이트에선 neutral.900 → 다크에선 neutral.50. 단, 다크모드는 단순 반전이 아니다: 채도를 약간 낮추고(눈부심 방지), 순수 검정(#000) 대신 약간 든 회색을 surface로 써 깊이를 살린다.

## 타이포 페어링

- **2폰트 원칙**: 디스플레이 1 + 본문 1이면 대개 충분. 모노는 코드/수치용 보조. 3개 초과는 산만.
- **대비를 만들되 충돌은 피한다**: 세리프 디스플레이 + 산세리프 본문(고전적 대비), 또는 같은 슈퍼패밀리의 다른 굵기(안전).
- **본문 가독성 우선**: 본문은 x-height 큰 산세리프, 16px 이상, 줄길이 45~75자.
- **시스템 폰트 폴백 스택**을 항상 함께 제시(웹폰트 로드 실패 대비).

## 시각 위계 = 절제된 대비

모든 것을 강조하면 아무것도 강조되지 않는다. 위계는 **소수의 강한 대비 + 다수의 절제**로:
- 크기(스케일), 굵기, 색 대비, 여백 중 **한두 개만 강하게** 쓴다.
- 1차 주목점 1개, 2차 2~3개, 나머지는 차분하게.
