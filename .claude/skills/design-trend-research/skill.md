# Design Trend Research — 트렌드를 원리로 변환하기

트렌드 조사의 목적은 베낄 대상이 아니라 **적용할 원리**를 얻는 것이다. "글래스모피즘이 유행"은 쓸모없고, "반투명 레이어로 깊이·위계를 만드는 기법"은 토큰 설계에 쓸 수 있다.

## 조사 절차

1. **축 고정** — design-director가 고른 디자인 언어 축을 받는다. 축이 명확하면 그 축의 동시대 사례에 집중하고, 미정이면 4축 전반을 얕게 훑어 director의 선택을 돕는다.
2. **다각도 수집** — 한 가지 검색으로는 한쪽만 본다. 최소 네 각도로 조사한다:
   - **제품 사례**: 실제 SaaS/앱/포트폴리오 (예: Linear, Vercel, Stripe, Arc 류)
   - **디자인 시스템**: 공개 디자인 시스템·토큰 (예: Radix, Material 3, Polaris)
   - **수상/큐레이션**: Awwwards, Godly, Mobbin 류의 큐레이션 흐름
   - **담론**: 디자인 글·발표에서 논의되는 "왜"
3. **원리 추출** — 각 사례에서 시각 요소가 아니라 **결정의 원리**를 적는다 (아래 표 참고).
4. **반트렌드 경고** — 접근성·가독성을 해치거나 곧 식상해질 클리셰를 표시한다.

## 표면 → 원리 변환 (핵심)

| 보이는 것(표면) | 추출할 원리(적용 가능) |
|---|---|
| "큰 헤드라인" | 타이포 대비 비율을 키운다 (스케일 ratio ↑, 본문/디스플레이 간격 ↑) |
| "여백이 많다" | 간격 스케일의 상단 단계를 적극 사용, 콘텐츠 밀도 ↓ |
| "은은한 그라데이션" | 서피스 레이어를 명도 2~4% 차로 분리해 깊이 표현 |
| "둥근/각진 모서리" | border-radius 토큰 스케일의 기조 결정 (0 / 모듈러 / 큰 pill) |
| "강렬한 한 색" | 무채색 베이스 + 단일 고채도 액센트 전략 |
| "두꺼운 보더·블랙" | 브루탈리즘: hard shadow, 1px→2~4px 보더, 시맨틱 대비 강화 |

## 출력 형식

`_workspace/01_trend_research.md`에 트렌드 항목별로:

```markdown
## [트렌드명]
- 설명: 무엇인가 (1~2줄)
- 원리: 토큰/시스템에 어떻게 환원되는가
- 레퍼런스: 제품/사이트명 + URL(출처)
- 적용 힌트(→geometry / →aesthetic): 어느 전문가가 어떻게 쓸지
- 주의: 접근성·수명 리스크
```

## 원칙

- **출처 없는 주장 금지.** 모든 발견에 레퍼런스 URL을 단다. WebSearch/WebFetch로 실제 확인한 것만 적는다.
- **적용 가능 수준까지 내린다.** "미니멀"에서 멈추지 말고 "1~2 폰트, 무채색+1, 간격 상단 단계, 큰 타이포 대비"까지.
- **director의 입력 재료일 뿐.** 트렌드는 방향을 잡는 데 쓰고, 최종 토큰 값은 geometry·aesthetic 전문가가 정한다. 트렌드를 그대로 토큰에 박지 않는다.
