# Grid Systems — 그리드와 수직 리듬

geometry-system 스킬의 보조 레퍼런스. 레이아웃 그리드를 정의할 때 로드한다.

## 8pt 그리드

모든 간격·치수를 8의 배수(4의 배수까지 허용)로. 이유: 대부분의 화면 밀도(@1x,@1.5x,@2x)에서 정수 픽셀로 떨어져 흐릿함이 없고, 디자이너·개발자가 암산하기 쉽다. 8(촘촘), 16(기본), 24, 32, 40, 48, 64...

**4pt 보조 그리드:** 아이콘 정렬·작은 컴포넌트 내부는 4px 단위가 필요할 수 있다. 8을 기조로 하되 4를 허용하는 하이브리드가 실용적.

## 컬럼 그리드

- **12 컬럼**: 2,3,4,6 분할이 모두 가능해 가장 유연. 데스크톱 표준.
- **거터(gutter)**: 컬럼 사이 간격. 간격 토큰에서 가져온다 (예: 24px).
- **마진(margin)**: 그리드와 뷰포트 가장자리 사이.
- **컨테이너 최대폭**: 가독성을 위해 본문 줄길이 45~75자(이상 66자) 범위가 되도록. 보통 max-width 1200~1280px.

## 브레이크포인트

콘텐츠가 깨지는 지점에서 정한다(기기명이 아니라). 흔한 출발점: 640 / 768 / 1024 / 1280 / 1536. 모바일 퍼스트로 작은 폭부터 정의.

## 수직 리듬 (Vertical Rhythm)

텍스트의 베이스라인이 일정 간격(baseline grid)에 맞아 떨어지게 하는 것. 

- 기준 라인높이를 정한다 (예: 24px = 8의 3배).
- 모든 텍스트 블록의 라인하이트와 상하 마진을 이 기준의 배수로.
- 헤딩은 라인하이트가 본문과 다르되, 차지하는 총 높이가 기준 그리드의 배수가 되도록 마진으로 보정.

효과: 컬럼이 여러 개여도 텍스트 줄이 가로로 정렬되어 페이지가 차분해진다.

## 비율 기반 레이아웃 분할

- 황금 분할: 1 : 1.618 (예: 본문 62% / 사이드 38%)
- 3분할 규칙: 시각 무게를 1/3 지점에 배치
- √2 중첩: 영역을 √2로 반복 분할하면 비율 보존

## 체크리스트

- [ ] 모든 간격이 기준 unit의 배수인가
- [ ] 컨테이너 폭에서 본문 줄길이가 45~75자인가
- [ ] 라인하이트가 baseline 그리드의 배수인가
- [ ] 브레이크포인트가 콘텐츠 기준으로 정해졌는가
