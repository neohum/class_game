# Modular Scale & 비율 이론

geometry-system 스킬의 보조 레퍼런스. 비율 선택이 헷갈리거나 스케일을 손으로 검증해야 할 때 로드한다.

## 모듈러 스케일이란

기준값(base)에 비율(ratio)을 반복 곱/나눗셈해 얻는 수열. `size(n) = base × ratio^n`. 음악의 음계처럼, 인접 단계의 비가 일정해 시각적 조화가 생긴다.

## 대표 비율

| 이름 | 값 | 출처/느낌 |
|---|---|---|
| minor second | 1.067 | 매우 촘촘, 미세 위계 |
| major second | 1.125 | 촘촘 |
| minor third | 1.2 | 차분, UI 본문에 흔함 |
| major third | 1.25 | 균형, 가장 범용 |
| perfect fourth | 1.333 | 또렷한 위계 |
| augmented fourth | 1.414 | √2, A4 용지 비율 |
| perfect fifth | 1.5 | 대담 |
| golden | 1.618 | 황금비 φ, 극적·상징적 |

## 황금비(φ)

φ = (1+√5)/2 ≈ 1.618. a/b = (a+b)/a 를 만족. 자연·고전 건축에서 발견되어 상징성이 크지만, **UI에서 φ는 종종 너무 큰 점프**를 만든다. 본문 주변 스케일엔 1.2~1.333이 실용적이고, φ는 디스플레이/히어로의 극적 대비나 레이아웃 분할(1 : 1.618 컬럼)에 쓰는 것이 안전하다.

## √2 시스템

1.414. 반으로 접어도 비율이 유지되는 성질(용지 규격). 그리드를 중첩 분할할 때 비율이 보존되어 레이아웃 시스템에 적합.

## 피보나치 간격

0,1,2,3,5,8,13,21,34,55... 인접 비가 φ에 수렴. 간격 스케일에 쓰면 작은 값은 촘촘하고 큰 값은 빠르게 벌어져, 컴포넌트 내부(작은 간격)와 섹션 사이(큰 간격)를 자연스럽게 구분한다.

## 손 검증 예 (major-third, base 16)

- 16 × 1.25 = 20
- 20 × 1.25 = 25
- 25 × 1.25 = 31.25
- 16 ÷ 1.25 = 12.8

스크립트 출력과 위 수동 계산이 일치해야 한다.

## 흔한 실수

- **두 개의 무관한 스케일.** 타이포는 1.25인데 간격은 임의 숫자(10,15,18) → 화음 깨짐. 간격도 기준 unit의 규칙적 수열로.
- **모든 단계를 다 씀.** 12단계를 만들어도 실제 UI는 5~7단계면 충분. 너무 많은 크기는 위계를 흐린다.
- **px 고정.** 접근성을 위해 rem 기반으로 두고 사용자 폰트 설정을 존중한다.
