# 게임 제작 기본 원칙 (GAME PRINCIPLES)

`public/games/` 아래에 새 게임을 만들거나 기존 게임을 수정할 때 **항상** 아래 원칙을 적용한다.
교실에서 전자칠판·태블릿으로 여러 학생이 동시에 쓰는 게임이라는 전제를 잊지 말 것.

## 1. 오른쪽 클릭 방지

- 게임 화면 전체에서 컨텍스트 메뉴가 뜨지 않아야 한다.

```js
document.addEventListener('contextmenu', (e) => e.preventDefault());
```

- 이미지·텍스트의 드래그/선택도 함께 막는다: `user-select: none;`, `-webkit-user-drag: none;`.

## 2. 멀티터치 — 꼬이지 않고 빠르게 반응

- **Pointer Events**(`pointerdown`/`pointerup`)를 사용하고, `click`·`touchstart` 혼용을 피한다.
- 터치 대상 요소에 `touch-action: none;`(또는 최소 `manipulation`)을 지정해 300ms 클릭 지연·스크롤 간섭을 없앤다.
- 여러 손가락이 동시에 눌러도 각 입력이 독립적으로 처리되어야 한다:
  - `event.pointerId` 단위로 상태를 추적하고, "마지막 터치 하나만" 처리하는 전역 플래그를 두지 않는다.
  - 한 플레이어의 터치가 다른 플레이어 영역의 입력을 막거나 취소하면 안 된다.
- 시각 피드백(눌림 효과)은 즉시(같은 프레임에) 반영한다.

## 3. 인원 구성 — 1 · 2 · 4 · 6인용

- 게임 시작 화면에서 **1, 2, 4, 6인**을 선택할 수 있게 만든다.
- 단, **원칙 6**(좁은 화면 판정)에 해당하는 게임은 1 · 2 · 3인까지만 제공한다.

## 4. 세로 배치 — 학년에 따른 컴포넌트 높이

- 기본: 게임의 모든 관련 컴포넌트(버튼·카드·입력 영역 등)는 **화면 세로 방향 중앙**에 배치한다.
- **1학년**: 키가 작아 화면 위쪽에 손이 닿지 않으므로 컴포넌트를 **화면 제일 하단**에 배치한다.
- 학년이 올라갈수록 배치 기준을 조금씩 위로 올려, 고학년에서는 세로 중앙에 오도록 한다.
- 구현 예 — 학년(1~6)을 설정으로 받아 세로 정렬 비율을 보간한다:

```js
// grade 1 → 화면 하단(1.0), grade 6 → 세로 중앙(0.5)
const anchor = 1.0 - (grade - 1) * 0.1; // 1.0, 0.9, 0.8, 0.7, 0.6, 0.5
```

- 학년 선택 UI가 없는 게임이라도 최소한 "저학년(하단)/고학년(중앙)" 선택은 제공한다.

## 5. 가로 배치 — 플레이어는 좌우로 나란히

- 1 · 2 · 4 · 6인 모드에서 각 플레이어 영역은 **가로로 나란히**(좌→우 1, 2, 4, 6개 컬럼) 배치한다.
- 위아래 2단으로 쪼개지 않는다. 각 컬럼은 같은 너비로 균등 분할한다.
- 각 플레이어 컬럼 안에서도 원칙 4(학년별 세로 위치)를 동일하게 적용한다.

## 6. 좁은 컬럼 판정 — 최대 3인으로 제한

- 4인·6인으로 가로 분할했을 때 **컬럼 너비가 너무 좁아 활동이 어렵다고 판단되는 게임**
  (예: 드래그 조작, 넓은 보드, 긴 텍스트 입력이 필요한 게임)은 4 · 6인 모드를 만들지 말고
  **최대 3인용**(1 · 2 · 3인)으로 만든다.
- 판단 기준 가이드: 1920px 전자칠판 기준 6분할 시 컬럼당 약 320px, 4분할 시 약 480px —
  이 너비 안에서 핵심 조작(터치 대상 최소 48px, 보드 전체 표시)이 성립하는지 확인한다.

## 7. 멀티플레이어 — 플레이어별 개별 시작 · 종료 · 다시 시작

- **2인 이상** 모드에서는 각 플레이어 컬럼이 **독립된 게임 인스턴스**여야 한다:
  - 각 컬럼에 자기만의 **시작 버튼**이 있어 준비된 학생부터 각자 시작할 수 있다.
  - 한 플레이어의 종료(완료)가 다른 플레이어의 진행을 멈추거나 방해하지 않는다.
  - 종료된 컬럼에는 **다시 시작 버튼**이 나타나 그 자리에서 혼자 재시작할 수 있다.
- 전체 동시 시작이 필요한 경쟁 게임이라도, 종료 후 **개별 다시 시작**은 반드시 제공한다.
- 구현 지침: 게임 상태(문제, 점수, 타이머, 진행 단계)를 전역이 아니라 **플레이어(컬럼) 단위 객체**로
  관리한다. 전역 상태는 인원 수 선택 정도만 갖는다.
