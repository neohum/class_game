# 전자칠판 게임 요청 양식 (GAME REQUEST TEMPLATE)

`public/games/`에 새 게임을 요청할 때 이 양식을 사용한다.
에이전트는 게임 제작 요청을 받으면 이 양식과 대조해 **빠진 필수 항목만** 되묻는다.

> **양식에서 묻지 않는 것** — 다음 공통 사항은 [GAME_PRINCIPLES.md](./GAME_PRINCIPLES.md)가
> 이미 고정하므로 요청자에게 다시 확인하지 않고 항상 적용한다:
> 오른쪽 클릭 방지, 멀티터치 즉시 반응, 1·2·4·6인 구성, 학년별 세로 배치, 플레이어 가로 분할.

## 필수 항목 (5개 — 빠지면 되묻는다)

| # | 항목 | 설명 |
|---|------|------|
| 1 | **게임 내용** | 무엇을 하는 게임인지 한두 문장으로. (예: "덧셈 문제를 풀어 로켓을 발사시킨다") |
| 2 | **규칙** | 진행 방식. 문제 제시 → 입력 → 정답/오답 처리 흐름, 오답 페널티 유무 등 |
| 3 | **대상 학년** | 1~6 중 하나 또는 범위. 원칙 4(세로 배치 기준)에 그대로 반영된다 |
| 4 | **승리/종료 조건** | 다음 중 하나 이상: 고정 문제 수 완료(타임어택) · 보드 완성 · 목표 점수 도달 · 목숨 소진(엔드리스) |
| 5 | **입력 방식** | **탭만** / **드래그 필요** 중 하나. 아래 자동 판정 규칙 참조 |

### 입력 방식 → 인원 자동 판정 (원칙 6)

입력 방식이 **드래그**이거나 **넓은 보드**(전체가 한눈에 보여야 하는 큰 격자, 긴 텍스트 입력)가
필요하면, 요청자에게 묻지 않고 [GAME_PRINCIPLES.md](./GAME_PRINCIPLES.md) **원칙 6**에 따라
**최대 3인(1·2·3인)** 구성으로 자동 판정한다. 탭만 쓰는 게임은 기본 **1·2·4·6인**.
경계가 애매하면(예: 드래그 대신 +/− 버튼 병행) 1920px 기준 6분할 컬럼 약 320px 안에서
핵심 조작(터치 대상 최소 48px, 보드 전체 표시)이 성립하는지로 판단한다.

## 선택 항목 (없으면 기본값 적용)

| 항목 | 기본값 | 설명 |
|------|--------|------|
| 교과 연계 / 학습 목표 | 없음 | 예: 수학(받아올림), 국어(어휘), 통합(분리수거) |
| 시간 제한 / 라운드 수 | **제한 없음 · 10문제** | 기존 게임 대다수가 10문제 타임어택(스톱워치 기록) 방식 |
| 학년별 난이도 범위 | 단일 난이도 | 예: 1학년=한 자리 덧셈, 3학년=받아올림 두 자리 |
| 소리 사용 | **음소거 시작 + 토글 버튼** | 소리를 쓰는 게임도 반드시 음소거로 시작하고 화면에 토글 제공 |
| 게임 이름 | 에이전트가 제안 | **한글 표시 이름** + **영문 폴더 id**(kebab-case, `public/games/<id>/`) |

## 빈 양식 (복사해서 사용)

```markdown
### 게임 요청
- 게임 내용:
- 규칙:
- 대상 학년:
- 승리/종료 조건:
- 입력 방식: (탭만 / 드래그 필요)
---
- 교과 연계/학습 목표: (선택)
- 시간 제한/라운드 수: (선택, 기본 제한 없음·10문제)
- 학년별 난이도 범위: (선택)
- 소리 사용: (선택, 기본 음소거+토글)
- 게임 이름(한글/영문 id): (선택)
```

## 작성 예시

기존 게임 24종을 기준으로 양식을 검증한 대표 예시 2건.

### 예시 1 — 탭 게임: 로켓 수학 (`rocket-math`)

- **게임 내용**: 받아올림·받아내림 연산 문제를 풀어 정답을 모으면 로켓이 발사된다.
- **규칙**: 문제와 보기 4개가 제시되고, 정답을 탭하면 로켓 게이지가 차오른다. 오답은 통과되지 않고 다시 고른다.
- **대상 학년**: 2학년
- **승리/종료 조건**: 정답 10개를 모으면 로켓 발사(완료), 걸린 시간 기록(타임어택).
- **입력 방식**: 탭만 → **1·2·4·6인** 구성.
- 교과 연계: 수학(받아올림·받아내림)
- 시간 제한/라운드 수: 제한 없음 · 10문제 (기본값)
- 소리 사용: 사용 안 함
- 게임 이름: 로켓 수학 / `rocket-math`

### 예시 2 — 드래그 게임: 분리수거 마스터 (`recycle`)

- **게임 내용**: 쏟아지는 쓰레기를 종류에 맞는 분리수거함에 넣는다.
- **규칙**: 화면의 쓰레기 아이템을 손가락으로 **드래그**해 5개 분리수거함 중 맞는 곳에 넣는다. 잘못 넣으면 되돌아온다.
- **대상 학년**: 1~2학년
- **승리/종료 조건**: 각자 보드의 아이템을 모두 올바르게 분류하면 완료, 점수 집계.
- **입력 방식**: 드래그 필요 → 원칙 6 자동 판정으로 **최대 3인(1·2·3인)** 구성.
- 교과 연계: 통합(바른생활 — 분리수거)
- 소리 사용: 음소거 시작 + 토글 (기본값)
- 게임 이름: 분리수거 마스터 / `recycle`

## 검증 노트 (기존 게임 24종 대조, 2026-07)

- 가장 흔한 패턴은 "탭 전용 · 10문제 타임어택 · 1·2·4·6인 선택"으로, 양식 기본값과 일치한다.
- `recycle`(드래그 → 1·2·3인)이 원칙 6 자동 판정의 실제 선례다. `clock-practice`(바늘 드래그)도 같은 판정 대상.
- 종료 조건은 고정 문제 수 / 보드 완성 / 목숨·점수 엔드리스 세 유형으로 모두 필수 항목 4의 보기에 포함된다.
- 기존 게임에는 음소거 기본+토글이 아직 없으므로(소리 쓰는 게임 4종 모두 즉시 재생), 이 양식의 소리 기본값은 **신규 게임부터 적용하는 규칙**이다.
- 폴더 id는 kebab-case를 표준으로 한다(`sound_plane`의 언더스코어는 예외적 과거 사례).
